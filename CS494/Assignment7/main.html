The Chinese Character Web API Documentation

What is it?

The Chinese Character Web API is a simple tool to get information about Chinese characters. There are many filtering options to limit by stroke count, radical, or to get Pinyin or definitions. It pulls data from the Unicode Standard Annex #38, Unicode Han Database, or Unihan for short.

The API is the work of Jens-Ingo Farley.

Why use it?

This tool is useful as a basis for another program - the creator, for example, has used the data from this tool to create a separate program to sort and view Chinese characters in much the same way. Classrooms or students of Chinese could also benefit from a quick way to look up characters, sounds, or meanings when the interest is more broad than definitions.

It was designed to be simple to use - while it is RESTlike, it simply uses JSON to return the data, which can then be displayed as-is or converted into romanized versions with the utility functions which apply diacritics for you.

Select the appropriate section above to learn about how to use the API. If you are new to this API, I suggest beginning at the first section and continuing on.

Structure of JSON Responses is an introduction to the sort of data that will be returned by various requests. As the fields are not entirely intuitive, it is a good idea to familiarize yourself with the fields you will most likely be using to avoid constantly looking up what the response will be.

Filtering Results goes over how to refine the responses to return the desired fields, such as pronunciations, strings, or definitions, or to filter the results by database section or character type.

Parsing and Displaying Data is a quick-start guide for using the data and displaying it in human-readable forms, and suggestions for how to utilize the data.

Finally, the Utility Functions provided by the creator are included for reference with brief explanation of what they do and their best uses.
